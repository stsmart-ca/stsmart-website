# STSmart.ca SEO Execution Plan (City Pages + Seller FAQs)

## Goal
Generate qualified seller leads for **multifamily** and **commercial** owners by ranking for city-intent and problem-intent searches.

## 1) Foundation (Week 1)
1. Measure baseline in Google Search Console + GA4:
   - current clicks/impressions for seller keywords
   - top landing pages
   - conversion rate by page
2. Build keyword buckets:
   - City + property type + intent (e.g., "sell multifamily building Toronto")
   - Pain points (vacancy, cap rate pressure, bad tenants, tax, timing, inherited property)
3. Define 8–15 priority cities first (don’t launch all cities at once).

## 2) Site Architecture (Week 1–2)
Create scalable page clusters:
- `/cities/[city]/`
- `/cities/[city]/multifamily/`
- `/cities/[city]/commercial/`
- `/faq/sell-multifamily/`
- `/faq/sell-commercial/`
- `/issues/[problem]/` (e.g., vacant-units, tenant-disputes, low-noi)

Internal linking rules:
- Every city page links to both property-type pages + 3 relevant issue pages.
- Every issue page links back to city pages and one conversion page.

## 3) City Pages Template (Week 2)
Each city page should include:
- Specific title tag and H1 with city + property type.
- 120–180 words about local market context (inventory, buyer demand, deal velocity).
- "Common owner concerns in [City]" section.
- Proof section (deals, anonymized case metrics, timelines).
- Clear CTA: request valuation / discuss exit strategy.
- FAQ block with 4–6 city-specific questions.

## 4) FAQ + Problem Content (Week 2–4)
Publish content around real seller questions:
- "When is the best time to sell a multifamily property in [City]?"
- "How do cap rates affect commercial sale price in [City]?"
- "Can I sell with difficult tenants?"
- "Should I renovate before listing a mixed-use building?"

Format for each page:
- Short direct answer first
- Detailed breakdown
- "What this means for your property" CTA block

## 5) On-Page SEO Standards (Ongoing)
For every new page:
- Unique title (50–60 chars), meta description (140–160 chars)
- Single H1, logical H2/H3
- FAQ schema where relevant
- LocalBusiness/Organization schema on core pages
- Image alt text with descriptive context
- Canonical + indexability checks

## 6) Local Authority + Trust (Week 3–6)
- Strengthen Google Business Profile categories and service areas.
- Build city-relevant citations and backlinks (local business orgs, CRE blogs, chambers).
- Add testimonials/case studies tagged by city and property type.

## 7) Conversion Optimization (Week 3–6)
- Dedicated CTA variants per page type (valuation, offer review, timing consult).
- Short form above the fold + sticky call button on mobile.
- Track form starts, submissions, calls by landing page.

## 8) 90-Day Rollout
- **Month 1:** 5 city hubs + 10 property-type pages + technical cleanup.
- **Month 2:** 15 issue/FAQ pages + internal linking pass.
- **Month 3:** 5–10 more cities + backlink/citation push + CRO tests.

## KPI Targets
- +40–60% impressions on city/property-intent queries
- +20–30% organic leads from non-branded search
- Top-10 rankings for priority city + intent terms in 3–6 months

## Step-by-Step Execution Checklist
1. Audit baseline performance.
2. Pick priority cities and keyword map.
3. Build page templates (city, multifamily, commercial, issue, FAQ).
4. Publish first cluster (5 cities + 10 subpages).
5. Add schema + internal links + conversion events.
6. Publish 2–3 FAQ/issue pages per week.
7. Review rankings/leads every 2 weeks; expand winning clusters.
